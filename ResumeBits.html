Mathulator

2010-2012

My first Java project and my first Android project, Mathulator is an Android application designed to one day replace modern graphing calculators. The reason the application was developed is because even in 2010 the idea of paying for an expensive graphing calculator was beginning to get ridiculous. More and more people were beginning to get mobile devices, and a mobile device could handle more powerful computation and give users a much better experience with graphs as they would have a touchscreen. Mathulator was built from the ground up, with every single line of code written by me including my own custom made mathematical computation library. As I learned more and more about Java and computer science in general I realized the design decisions I made in high school, with no computer science education, were non optimal, and the entire application had to be refactored. The new design for the application was 10x as powerful and likely would have removed much of the bottlenecks the application faced. The newly designed application however, is too much for one person to develop, so the project’s development halted.

Features

  • Standard mathematical expression evaluation
  • Standard set of mathematical functions (sin, cos, abs, ln ect..)
  • Numerical derivative evaluation
  • Numerical integral evaluation
  • Interactive multi function graphing
  • Interactive graphed numerical derivatives
  • Interactive graphed numerical integrals
  • Autocomplete like system for variables and functions when typing
  • Analytics reporting statement evaluation times and user engagement

Light Droid

2014

Developed with 3 other students as a senior project for a security company called Lumension, Light Droid is designed to scan through all applications installed on an Android device as well as scanning device level settings to warn the user about potential security issues. The reason this application was developed is because Android users typically have several applications installed, and checking the permissions used by every application can be a daunting task, as well as the fact that most users aren’t aware of security threats such as Wifi spoofing. My part of the development project was building the interface, maintaining proper Java standards on all committed code, and validating the threat detection algorithms. The application’s development ended at the end of the semester, though further development of it may continue as a part of future senior projects.

Features

  • Scans all installed applications and rates their possible threat by comparing their permissions and Google Play download count and ratings
  • View more information about an application than is usually easily accessible to the user
  • Filter ratings quickly by name
  • Scan device level settings and rate the device overall while warning the user about potential security threats
  • Manually adjustable weights for each checked permission
  • Blacklist/Whitelist applications to automatically be marked as a high level threat or removed entirely from checks.

Project Resume

Project Resume was a quickly put together project designed to address all of the flaws in paper resumes or resumes as a sharable document (such as a Google Doc). The application was build from the ground up in 8 days, and uses modern design principals in both application and user experience design, as well as the latest in technologies.

The problems with paper resumes

  • Sharing a piece of paper is hard
  • You either have to scan it and email it, or copy it and physically pass it around.
  • Managing custom resumes can become ridiculous
  • As you build customized resumes for more and more potential employers, managing each document becomes harder and harder.
  • You usually end up with a resume that only has different wording in some places, otherwise properly customizing 20 resumes would take weeks.
  • This is also true of resumes as a sharable document
  • Paper resumes expire
  • When a potential employer holds on to a resume for any period of time the resume can quickly become out of date, and potential employers can miss out on new and important information.
  • Paper resumes don’t show developers off
  • A graphical designer can build a really cool and original resume that shows of how good they are at their trade. But for almost every other field a resume is all talk, and does nothing to show how you handle your trade.
  • This is also true of resumes as a sharable document

Design

The design goal of this application was for me to, basically, show off. Project Resume was designed from the ground up using the most modern technologies I was aware of, and as such, Project Resume was not only my first real Go project, but also my first Polymer project, and my first AppEngine project.

The frontend

The frontend, which you are using right now, was built using a Polymer, html5, css3, and Javascript. The frontend speaks with an AppEngine application over a CloudEndpoints based REST service to automatically load the resume’s data from the Resume Services based off of the AccessKey given when the front end first loads. The AccessKey can be given as a URL parameter , or entered when the front end loads.

The Backend

The backend is several CloudEndpoints based REST Apis written in Go. Data objects are stored in Cloud Datastore with an access control list that defines whether or not each item should be shown to a viewer with a specific access key. The service then filters out all of the requested data items and exposes them to the client while stripping the access control list so that the client can’t see other parties AccessKeys.

Designing the application in this way allows the resume to automatically customize itself based on the AccessKey provided by hiding objects, and then showing different versions of objects based on the access control list.

UX Design

I am not a User Experience designer. However, I know that a delightful user experience is important in any application. That is why Polymer was chosen for the interface, Polymer makes designing interface that fit the Material Design language very easy.
